# 검색 재정렬(rerank) 프롬프트 — 질의-자기표현 매칭도(정렬용)

`RERANK_MODE=llm` 일 때 `retrieve_hospital` 의 1차 후보(임베딩+주력강도 정렬)를
LLM 으로 한 번 더 정렬하는 2-stage RAG 의 2단계용. 출력은 **목록 순서를 정하는 내부
매칭 점수**일 뿐 사용자에게 노출되지 않는다(SearchResult 는 metadata-only, 의료법 §56③ 유지).

## 의료법 — 절대 어기지 말 것

- 이건 병원의 **우수성·추천·등급 판단이 아니다.** "이 병원이 자기 표현 자료(자칭·블로그·
  후기·사진 발췌)에서 질의 주제를 얼마나 **다루는가**"의 *목록 정렬용 매칭도*만 매긴다.
- "잘한다 / 추천 / 명의 / 전문 / 탁월 / 우수" 같은 평가·광고 어휘를 **생성하지 말 것.**
  너는 정렬용 점수 숫자만 낸다(자연어 코멘트·등급 라벨 금지).
- 발췌는 그 병원이 **스스로** 공개한 자기표현이다. 너의 임상 지식으로 병원을 평가하지
  말고, 오직 "이 자기표현 텍스트가 질의 주제와 얼마나 맞물리는가"만 본다.

## 점수 기준 (0.0 ~ 1.0, 순서 보정용 — 병원 등급 아님)

질의 주제와 그 병원의 **자기표현이 맞물리는 정도**를 매긴다:

- **0.8~1.0**: 발췌·자칭 주력이 질의 주제를 *주력/핵심*으로 반복해 드러냄.
- **0.4~0.7**: 질의 주제를 다루지만 여러 진료 중 하나거나 부수적.
- **0.0~0.3**: 질의 주제가 1회 스침·맥락만 겹침·사실상 무관.

질의가 질환(예: 독감·무좀·편도염)인데 발췌가 미용·성형 위주면 낮게.
질의 주제를 자기표현에서 *명시적으로 주력*이라 밝힌 병원일수록 높게(병원의 질이 아니라
질의-자기표현 매칭의 강도다).

## 입력

사용자 질의:
{query}

후보 병원 (각 줄: `[번호] 병원명 | 자칭 주력 | 자기표현 발췌`):
{candidates}

## 출력 — 엄격한 JSON 한 개만

모든 후보 번호에 대해 빠짐없이 점수를 매긴다. 코드블록·설명 없이 JSON 만:

```json
{"scores": [{"i": 0, "s": 0.0}, {"i": 1, "s": 0.0}]}
```

- `i`: 입력의 후보 번호(정수).
- `s`: 0.0~1.0 매칭도 점수(소수, 정렬용).
- 후보 수와 출력 `scores` 길이가 **반드시 일치**.
