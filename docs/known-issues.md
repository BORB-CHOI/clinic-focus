## ISSUE-001 · 후기 시그널의 감성·부정(negation) 미처리

- **상태**: 🟡 미해결 (추후) — 당장 터지는 버그 아님, 검색 관련성 오염 가능성
- **발견일**: 2026-05-31
- **트랙**: ai/ (signal-fusion / vector-search)
- **우선순위**: 중 (PoC 시연엔 수용 가능, 정식화 전 결정 필요)

### 증상 / 시나리오

네이버 후기 중 이런 **부정 후기**가 있다(실제 표본 발췌, 요약):

> "당화혈색소 건강검진 기준치 넘어가서 3개월 당관리하고 검사받으러 갔는데 …
> 기준치가 잘못됐다며 **검사 거절당함**. … 정말 불쾌했습니다. **다신 안가요**"

병원이 그 서비스를 **거절했고** "안 어울린다"는 내용인데, 파이프라인이 "당화혈색소·건강검진"
같은 토픽을 **긍정 신호처럼** 다뤄 그 병원을 해당 토픽 검색에 노출시키지 않는가 — 가 우려.

### 근본 원인 (코드 판단)

파이프라인 어디에도 **감성(sentiment)·부정(negation) 판정이 없다.** 두 경로로 나뉜다:

**1) 분류 시그널 경로 — 이 예시는 우연히 안전**
- 후기 키워드 추출 [`ai/pipeline/classify.py:288`](../ai/pipeline/classify.py#L288) `_count_medical_keywords`
  는 **고정 화이트리스트** [`_KEYWORD_TO_FOCUS` (classify.py:108)](../ai/pipeline/classify.py#L108) 의
  단어만 센다. 현재 4개 과(피부·정형·이비인후·안과)만 모델링 → "당화혈색소"·"건강검진"(내과 계열)은
  사전에 없어 **추출 자체가 안 됨** → `primary_focus` 0 기여.
- 즉 이 예시가 안전한 건 *감성을 봐서가 아니라* 그 토픽이 사전에 없어서다(우연). 사전에 있는
  단어라면(예: "보톡스 부작용 최악") 부정 후기라도 "보톡스" +1 → "미용 시술" focus 에 투표한다.
  (단, 이 시스템은 "잘한다"가 아니라 "무엇을 **하는가**"를 측정 → 토픽-빈도=focus 는 부분적으로 정당.)

**2) KB 임베딩 경로 — 여기서 우려가 실재**
- [`ai/search/kb_store.py:319-331`](../ai/search/kb_store.py#L319) `build_reviews_chunk` 가
  **후기 본문 원문을 통째로** reviews 청크에 넣어 Titan v2 로 임베딩한다.
- 임베딩은 **감성을 구분 못 한다.** 부정 후기라도 "당화혈색소 건강검진" 토큰이 본문에 있으면,
  사용자가 "당화혈색소 검진 잘하는 곳"으로 검색할 때 **검사를 거절당한 병원이 의미 유사로
  딸려 나올 수 있다.** ← 우려의 실체.
- 가장 고약한 케이스: "이 병원은 X를 안 한다 / 거절했다" 후기가 (X가 사전 단어면) 도리어 X focus
  로 투표하는 **인버전**. negation 감지가 0이라 못 거른다.

### 현재 안전장치 (그래서 당장 치명적이진 않음)

| 안전장치 | 근거 |
|---|---|
| 후기 원문은 화면에 **절대 미표시** (임베딩 전용) | [`kb_store.py:828`](../ai/search/kb_store.py#L828) `_raw_to_search_result` 가 metadata 만 반환 (의료법 §56③) |
| 검색 = 생성형 답변 아닌 **정렬된 목록** — "잘함"이라는 문장 생성 LLM 없음 | [`ai/CLAUDE.md`](../ai/CLAUDE.md) "검색 시점 LLM 0건" |
| 분류 설명(`generate_description`)도 후기 raw 미참조 | [`classify.py:417-419`](../ai/pipeline/classify.py#L417) `ReviewSignal` 은 키워드 빈도·토픽만 보유, 본문 raw 미저장 |

→ "부정 후기가 긍정 평가로 화면에 노출"되는 일은 **없다.** 피해는 **검색 후보 목록에 토픽 유사로
섞이는** 관련성 오염에 한정.

### 해결 방향 (택1 또는 조합)

1. **임베딩 전 후기 감성/거절 게이트** — `build_reviews_chunk` 앞단(또는 `_analyze_reviews`)에서
   강한 부정·"취급 안 함"·"거절"·"안 가요" 류 후기를 **제외하거나 감점**. 정공법.
2. **negation 룰** (저비용) — 토픽 토큰 주변 부정어("안", "못", "거절", "없", "취급 안") 윈도우
   탐지로 해당 토픽 기여 무효화. 1만 룰 베이스라인에도 적용 가능.
3. **LLM 감성 판정** — 비용 때문에 1만 풀커버엔 부적합하나, **LLM 시연 10개엔 붙일 만**.

### 결정 필요 사항

- 1만 룰 경로에도 negation 룰(저비용)을 넣을지, 아니면 시연 10개만 감성 처리할지.
- 후기 청크에서 부정 후기를 **제외**할지 **감점 보존**할지(임베딩 어휘 풍부도 vs 관련성).
- 처리 시 의료법 재검수 필요: 청크 구성 변경은 [`kb_store.py:247-251`](../ai/search/kb_store.py#L247)
  §56③ 면제 조건과 충돌하지 않는지 `medical-language-reviewer` 검수.
