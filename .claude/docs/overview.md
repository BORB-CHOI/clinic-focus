# 병원 검색 서비스 기획안

## 0. 한 줄 정의

> **표준 진료과목 카테고리 너머, 병원이 실제로 무엇에 집중하는지를 알려주는 검색 서비스.**

---

## 1. 문제 정의

### 핵심 통점 — 카테고리 거짓말

환자는 "피부과 = 모든 피부 질환을 본다"고 믿고 간다. 그러나 실제 그 피부과는 "피부 클리닉(미용) 전문"이라 아토피·여드름 같은 일반 피부질환은 사실상 보지 않는다. 환자는 표준 카테고리를 신뢰한 대가로 시간·교통비·심리적 비용을 손해 본다.

### 실제 사례

- **세부 전문성 미스매치**: 정형외과·이비인후과·피부과·안과 모두 같은 카테고리 안에서 의사마다 주력 분야가 다름. 환자는 그것을 알 수 없음.
- **기계적 처치**: 간호사가 방문 목적만 듣고 치료실로 보내 기계적 처치만 받는 경험.
- **잘못된 안내**: 탈모는 치료가 불가능하고 예방만 가능하다며 약국 권유 → 약국에서 받은 판시딜은 모근 영양제로 원형 탈모용. M자 탈모는 병원 처방이 필요했음.
- **장비 부재**: 사마귀 냉동 치료 장비 없음. 10초 라식 기기 없음. 헛걸음.
- **피부 클리닉 전문 피부과**: 일반 진료를 거의 보지 않는데 외관은 일반 피부과로 보임.
- **불쾌한 진료 경험**: 마케팅을 전혀 하지 않는 오래된 의사의 고집스러움·무례함.

### 사회적 검증

SNL 코미디에서 환자가 아토피 때문에 피부과를 찾아갔는데 "피부 클리닉 전문"이라 손님을 돌려보내려 하고, 환자가 "피부과인데 왜 아토피 하나 못 보냐"고 받아치는 장면이 유행. **이 통점이 이미 대중적 공감대를 확보했다는 외부 증거**.

## 2. 타겟 사용자

- **1차 메인 타겟**: 전 국민. 표준 진료과목으로 분류된 동네 의원을 이용하는 모든 사람.
- **초기 진입 그룹**:
  - 동네 의원 헛걸음 경험자 (1회 이상 진료 거절·미스매치 경험)
  - 부모·맞벌이 가구 (자녀 진료 시 카테고리 함정 피해 큼)
  - 만성질환자·재진 환자 (의사 전문성에 민감)

**비급여 시술 한정 타겟이 아님.** 일반 진료를 포함한 전 국민이 비전 좌표.

### 4-2. 분류 체계 — 카테고리 이중 색인

기존 표준 진료과목 위에 **"실제 주력 분야"** 태그를 추가로 부착.

```
피부과
  ├─ 실제 주력: 미용 시술 (보톡스·필러·레이저)
  ├─ 실제 주력: 일반 진료 (아토피·여드름·습진)
  └─ 실제 주력: 피부암·종양

정형외과
  ├─ 실제 주력: 척추
  ├─ 실제 주력: 어깨·견관절
  ├─ 실제 주력: 무릎·관절
  └─ 실제 주력: 손·발 (수부외과)

이비인후과
  ├─ 실제 주력: 알레르기·비염
  ├─ 실제 주력: 청각·이명
  └─ 실제 주력: 코·수면호흡
```

### 4-3. 검색 인터페이스

두 가지 진입 방식을 모두 지원한다.

**(1) 자연어 검색**
- 예: "M자 탈모 처방받을 수 있는 강남 의원" → 키워드 단순 매칭 아닌 RAG 의미 검색
- 검색창 한 줄에서 시작

**(2) 지도 기반 내 근처 검색** ⭐
- 사용자 현재 위치(GPS) 또는 지정 위치 기준 반경 내 병원 표시
- 카카오맵 또는 OpenStreetMap 임베드
- 반경 조정: 0.5km / 1km / 3km / 5km / 10km
- 지도 마커 색상이 신뢰도 등급별로 다름 (확실 / 추정 / 정보 부족)
- 마커 클릭 시 카드 팝업 → 상세 페이지 진입
- 자연어 검색 결과 + 지도 토글 동시 지원 (예: "어깨 잘 보는 정형외과" 검색 후 지도 뷰로 전환)
- 결과 리스트는 거리순 / 신뢰도순 정렬 토글

**검색 결과 카드 구성** (요약 노출):
- 표준 진료과목 + 실제 주력 분야 태그
- 신뢰도 점수 + 등급 (확실 / 추정 / 정보 부족)
- 한 줄 요약 (예: "일반 피부 진료 중심, 미용 시술은 거의 안 하는 동네 의원")
- 현재 위치 기준 거리 (지도 검색일 때)

### 4-4. AI 통합 상세 설명 — **본 서비스의 진짜 차별점**

검색 결과를 클릭해 진입하는 **병원 상세 페이지가 본 서비스의 핵심 결과물**이다. AI 자연어 설명은 그 페이지의 헤드라이너이고, 그 아래에 9개 영역의 풍부한 정보가 받쳐준다. 단순 태그·평점·후기 나열이 아니라 환자가 헛걸음 없이 결정할 수 있을 만큼의 종합 정보를 한 페이지에 응축한다.

**기존 서비스의 한계** (굿닥·모두닥·똑닥·어디아파 공통):
- 위치·전화·운영시간·의사 약력을 정보 조각으로 늘어놓음
- 사용자 리뷰를 시간순·평점순으로 나열
- "이 병원이 다루지 않는 분야"는 어디에도 표시하지 않음 → 헛걸음 발생
- "이 병원이 어떤 곳인지"의 통합 해석은 사용자가 직접 해야 함

---

### 상세 페이지 9개 영역 구성

#### ① 헤드라이너 — AI 통합 설명
4 시그널(자칭 컨셉 / Vision / 블로그 / 후기) 데이터를 Bedrock Claude Sonnet 4.5에 입력해 병원의 정체성·실제 진료 영역·강점·약점·주의사항을 자연어 단락으로 생성. 모든 문장에 출처 시그널 배지(`[사이트]` `[Vision]` `[블로그]` `[후기]`)를 부착해 사용자가 근거를 즉시 추적 가능. 신뢰도 점수와 등급도 같이 노출.

**예시 출력**:

> **○○피부과는 일반 피부 진료 중심의 동네 의원입니다.**
>
> 홈페이지 메인 화면에서 아토피·여드름·습진 같은 일반 피부질환을 가장 먼저 안내하고 있으며 [사이트], 시술 사진 80%가 일반 진료 케이스고 미용 시술 사진은 18%로 적습니다 [Vision]. 블로그 글 50건 중 아토피 관련 글이 34%, 여드름 관련 글이 21%로 가장 많고, 미용 시술 관련 글은 5건뿐입니다 [블로그].
>
> 실제 방문 후기에서도 "친절한 아토피 상담", "꼼꼼한 여드름 치료" 같은 키워드가 자주 등장합니다 [후기]. 다만 사마귀 냉동치료기·점 제거 레이저 같은 시술 장비는 보유하고 있지 않은 것으로 보이므로, 미용 목적이라면 다른 병원을 권합니다.
>
> *분류 신뢰도: 92% — 사이트 메인 35% + 시술 사진 28% + 블로그 17% + 후기 12%*

#### ② 핵심 진료 정보
- **표준 진료과목 + 실제 주력 분야 태그**
- **세부 진료 항목 리스트** — 이 병원이 실제로 다루는 증상·시술. 예: "아토피 / 여드름 / 습진 / 사마귀(냉동치료) / 점 제거(레이저)"
- **다루지 않는 분야 명시** ⭐ 헛걸음 방지의 핵심 — 예: "M자 탈모 처방 ✗ / 보톡스·필러 ✗ / 점 제거 레이저 ✗"
- **보유 의료기기** — Vision + 공공 신고 기반. 예: "냉동치료기 ○ / 더모스코프 ○ / 점 제거 레이저 ✗"
- **비급여 항목 가격** (사이트에 공개된 경우만)

#### ③ 의료진 정보
- 원장·의사 명단
- 전문의 자격 (심평원 공공 데이터)
- 세부 전공·경력 — 사이트에서 추출. 예: "서울대 의대, 어깨 관절 세부전공"
- 의사별 진료 분야가 다르면 분리 표시

#### ④ 신뢰도·근거 영역
- 신뢰도 게이지 + 4 시그널 기여도 분해 차트
- 펼침 메뉴: **자칭 컨셉 근거** — 사이트 원문 인용 + 추출 키워드 + 원문 URL
- 펼침 메뉴: **시술 사진 분포** — Vision 분석 결과 + 대표 이미지 썸네일
- 펼침 메뉴: **블로그 주제 분포** — 상위 토픽 + 글 개수
- 펼침 메뉴: **후기 키워드 빈도** — 상위 키워드 + 후기 N건 기반
- **공공 데이터 출처 표시** — 심평원 등록 정보 링크

#### ⑤ 기본 운영 정보
- 주소 + 지도 (카카오맵 또는 OpenStreetMap 임베드)
- 전화번호 (모바일에서 탭하면 전화 연결)
- 운영시간 — 요일별, 점심시간 분리
- 야간·주말·공휴일 진료 여부
- 주차 가능 여부
- 예약·접수 방법 (현장 / 전화 / 온라인)
- 홈페이지 링크

#### ⑥ 사용자 피드백 영역
- **분류에 대한 1-tap 피드백** (👍/👎) — 익명, localStorage 디바이스 ID 기반 1회 제한
- **신뢰도 누적 통계** — "이 분류에 대한 사용자 피드백: 👍 87% / 👎 13% (N=145)"
- **분류 오류 신고 버튼** — 인간 검수자 큐로 전달

#### ⑦ 분류 변경 이력 미리보기
- 최근 변경 1~2건 (예: "2026.04 미용 전문 → 일반 진료 비중 증가로 변경")
- 변경 사유 (피드백 누적 / 인간 검수 / Vision 재분석)
- 전체 이력 페이지 링크

#### ⑧ 관련 병원 추천
- 같은 동네에서 **같은 세부 주력**을 다루는 다른 병원 3~5곳 (S3 Vectors 유사도)
- **이 병원이 안 다루는 분야의 대안 병원** — 예: "사마귀 냉동치료는 이 병원에 없으므로, 동네에서 다루는 곳: △△의원, □□피부과"
- 헛걸음 방지의 마지막 안전망

#### ⑨ 메타 정보
- 데이터 마지막 업데이트 시각
- 데이터 출처 고지: "이 정보는 병원 홈페이지·공공 데이터·사용자 후기에서 자동 수집된 정보를 AI가 정리한 결과입니다"
- 분류 신뢰도가 70% 미만이면 "정보 부족 — 직접 병원에 문의 권장" 경고 배너

### 데이터 출처 매핑표

각 영역이 어디서 오는 데이터인지 명시. 의료법 회색지대 회피의 기반.

| 영역 | 데이터 출처 |
|---|---|
| ① AI 통합 설명 | 최비성 `generate_description` (4 시그널 종합) |
| ② 세부 진료 항목 | 자칭(LLM 추출) + Vision + 블로그 |
| ② 다루지 않는 분야 | 자칭 + Vision 부재 + 후기 부재 → 추론 |
| ② 보유 의료기기 | Vision + 심평원 공공 신고 |
| ② 비급여 가격 | 사이트 크롤링 (있을 때만) |
| ③ 의료진 약력 | 사이트 크롤링 + 심평원 전문의 자격 |
| ④ 신뢰도·근거 시그널 | 최비성 4 시그널 |
| ⑤ 주소·전화·운영시간 | 심평원 공공 데이터 + 사이트 크롤링 |
| ⑥ 사용자 피드백 통계 | DynamoDB 집계 |
| ⑦ 분류 변경 이력 | DynamoDB 이력 테이블 |
| ⑧ 관련 병원 추천 | S3 Vectors 유사도 검색 |

### 의료법 회색지대 회피 — 주체 명시 표현

전 영역에서 동일하게 적용. "이 병원은 ~ 잘한다"가 아니라 **"이 병원이 자기 사이트에서 ~를 메인으로 표시함"** 형태로 항상 출처를 밝힌다. 우리는 평가하지 않고, 병원이 자기 자신을 어떻게 표현했는지를 종합해서 보여줄 뿐. 6-1 표현 가이드라인을 모든 영역에 일관 적용.

## 5. 핵심 알고리즘

### 5-1. 자기 파괴 방지 — 4 시그널 교차 검증

자칭 컨셉만 색인하면 병원이 "전문" 키워드를 도배해서 신호가 무력화됨. 4개 시그널을 교차 검증하여 도배 공격 차단.

| 시그널 | 소스 | 가중치 |
|---|---|---|
| 1. 자칭 컨셉 | 사이트 메인 페이지·소개 텍스트 | 25% |
| 2. 시술 사진·의료기기 | AI Vision 이미지 분석 | 30% |
| 3. 블로그 주제 분포 | 포스팅 키워드 빈도 | 20% |
| 4. 후기 키워드 | 사용자 후기·공공 데이터 | 25% |

**검증 로직**:
- 4개가 한 방향으로 정렬 → 신호 강도 95%↑ ("확실")
- 일부 정렬 → 70~95% ("추정")
- 자칭만 강하고 나머지가 다른 방향 → **자칭 도배 의심 페널티**

### 5-2. AI 분류 신뢰도 시스템

의료 도메인 오분류는 환자 피해로 직결되므로, 정확성보다 **투명성**으로 신뢰 확보.

- **신뢰도 점수 노출 의무화**: 분류 결과 옆에 항상 신뢰도 점수와 근거 시그널을 함께 노출. 사용자가 직접 판단 가능.
- **사용자 피드백 루프**: 검색 후 방문한 사용자에게 1-tap 질문 "이 병원이 실제로 ○○ 분야를 잘 봤나요?" → 피드백 N건 이상 누적 시 분류 보정.
- **오류 신고 즉시 검수**: 환자가 잘못된 분류 신고 → 인간 검수자가 24시간 내 검토. 의료 도메인 전문가 자문 패널 운영.
- **분류 변경 이력 공개**: 분류가 바뀌면 변경 이력 공개. "이 병원은 2026년 3월까지 미용 전문으로 표시됐다가, 일반 진료 비중 증가로 4월 변경됨".

## 6. 의료법 대응 — 주체 명시 원칙

> **핵심 원칙: 우리가 평가하지 않는다. 병원이 자기 자신을 어떻게 표현했는지를 보여줄 뿐이다.**

의료법 제56조①에 의해 의료기관이 아닌 자는 의료광고를 할 수 없음. **우리는 광고주가 아닌 정보 제공자**라는 정체성을 표현에서 일관되게 유지.

### 6-1. 표현 가이드라인

| 잘못된 표현 (광고 해석 위험) | 적법 표현 (정보 제공) |
|---|---|
| "이 병원은 미용 전문이라 아토피 환자는 가지 마세요" | "이 병원이 자기 홈페이지에서 미용 시술을 메인으로 표시함" |
| "어깨 잘 보는 정형외과" | "이 병원이 자기 사이트에서 어깨 관절을 주력으로 소개함" |
| "여기 사마귀 냉동치료기 있음" | "이 병원이 공식 신고한 의료기기 목록에 냉동치료기 포함됨" |
| "이 의사는 탈모 처방을 잘함" | "이 의사가 자기 블로그에서 M자 탈모 처방 사례를 다룸" |

### 6-2. 출처 명시 의무화

모든 노출 정보에 출처 태그:
- `[병원 홈페이지]` — 사이트 텍스트 출처
- `[공공 신고]` — 심평원 등 공공 데이터 출처
- `[사용자 후기 N건]` — 후기 키워드 빈도 출처

### 6-3. 환자 치료 경험담 활용 금지

후기는 평가·추천이 아닌 **키워드 빈도 데이터로만** 활용. "이 병원에서 아토피 치료받고 좋았어요" 같은 개별 후기는 노출하지 않고, "아토피·여드름 키워드가 후기에 60% 등장" 같은 통계만 활용.

### 6-4. 법적 준비

- 변호사 자문 완료 + 자문 의견서 보관
- 보건복지부 사전 질의 회신 확보 (PoC 단계에서)
- 의료광고 사전심의 대상 여부 정기 모니터링

## 8. 수익 모델 — 3중 구조

비전 유지(일반 진료 포함, 전 국민 대상)를 위해 광고 모델 회피. 광고에 의존하면 모두닥처럼 비급여 분야로 끌려가게 됨.

### 8-1. 1차 — B2B 라이선스 (메인 수익)

- **보험사 부가 서비스 라이선스**
  - 실손보험사 가입자 앱에 "내 증상 → 맞는 병원" 검색 기능 탑재
  - 가입자당 연 1,000~5,000원 (보험사 부가 서비스 단가 추정)
  - 가입자 100만 명 보험사 1곳당 연 10~50억 매출
- **기업 복지몰**
  - 대기업·중견기업 임직원 헬스케어 솔루션
  - 인당 월 단위 과금
  - 1만 명 기업 1곳당 연 2~5억
- **의료기기·제약사 시장 데이터 판매**
  - "충청권 정형외과의 어깨 전문 표방 비율" 같은 시장 인사이트 보고서
  - 광고가 아닌 데이터 라이선스라 의료법 회피
  - 보고서당 수천만~수억

### 8-2. 2차 — 공공·사회적 수익

- 보건복지부·심평원 시범사업·연구용역 수주 (연 5~20억)
- 사회적 기업 인증 → 임팩트 투자 유치 (소풍벤처스·MYSC 등)
- 지자체 시니어 헬스케어 사업 제휴

### 8-3. 3차 — 진료 매칭 수수료

- 예약·접수 매칭 (똑닥 합법 모델)
- 비대면 진료 연계 매칭 (닥터나우·솔닥·올라케어 제휴)

### 핵심 원칙

> **사용자(환자)는 영원히 무료. 수익은 B2B와 공공에서 받는다.**

## 13. 최종 메시지

이 서비스의 본질은 **표준 진료과목 카테고리가 환자를 속이는 구조를 폭로하는 것**이다. 기존 모든 병원 검색 앱은 표준 카테고리를 신뢰하는 전제 위에 서비스를 쌓았다. 본 서비스는 그 전제가 거짓이라고 폭로한다.

**본업은 태깅이 아니라 설명이다.** 4 시그널을 수집해 태그를 다는 것은 검색·필터 보조용이고, 진짜 결과물은 병원 상세 페이지에 출력되는 AI 자연어 통합 설명이다. 굿닥·모두닥·똑닥은 정보 조각을 늘어놓고 해석은 사용자에게 떠넘긴다. 본 서비스는 그 해석을 AI가 미리 해서 자연어로 풀어 쓴다.

수익 모델은 광고가 아니라 B2B 라이선스 + 공공 사업 + 매칭 수수료의 3중 구조이며, 이로써 비전(일반 진료 포함, 사용자 무료, 비급여 한정 X)을 1mm도 양보하지 않는다.

법적 안전성은 주체 명시 원칙으로 확보한다. 우리는 평가하지 않고, 병원이 자기 자신을 어떻게 표현했는지를 그대로 보여준다.

AI 분류의 신뢰성은 정확성보다 투명성으로 확보한다. 신뢰도 점수와 시그널 근거를 항상 함께 노출하고, 사용자 피드백으로 보정한다.

SNL이 사회적으로 검증한 통점 위에서, 굿닥·모두닥이 6년간 풀지 못한 카테고리 함정을 해결한다.
